Logarithm base in algorithm complexity

2013-09-15

I was reading up on algorithm complexity and found the following statement: within the context of algorithm complexity, the base of a logarithm is irrelevant.

This was not demonstrated in any way and the reader was obviously expected to accept this as obvious - which is the case, unless your maths are as rusty as mine.

Constants in algorithm complexity

The first step to enlightenment is to remember that constants are considered irrelevant when expressing an algorithm’s complexity: O(2N) is always written O(N).

With that in mind, we “only” need to prove that logarithms with different bases are linked together by a constant:

∀i, j ∃cij ∣ ∀x logi(x) = cij * logj(x)

Linking two logarithms with different bases

In order to find this cij constant, let’s start from the definition of a logarithm:

y = logi(x) ⇔ x = iy ⇔ logj(x) = logj(iy) ⇔ logj(x) = y * logj(i)

Replacing y by its actual value, we get:
logj(x) = logi(x) * logj(i)

Which allows us to state what we set out to prove:
∀i, j, x logj(x) = logj(i) * logj(x)

That is, since there always exists a known constant that links two logarithms in different bases, and constants should be ignored when expressing an algorithm’s complexity, then the base of a logarithm is irrelevant within the context of algorithm complexity.
